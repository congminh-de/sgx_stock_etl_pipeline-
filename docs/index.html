SGX Derivatives ETL-Analytics Pipeline

End-to-end automation for analyzing market microstructure, tracking institutional capital flow, and visualizing intraday volatility patterns on the Singapore Exchange.

The Business Challenge

The Singapore Exchange (SGX) derivatives market generates massive volumes of high-frequency tick data every trading day. For quantitative analysts and institutional traders, extracting meaningful signals from this noise is mission-critical.

The core requirement was to ingest daily transaction logs to analyze market microstructure, specifically tracking contract volatility, hourly turnover velocity, and identifying significant volume spikes that indicate institutional activity.

The significant constraint was infrastructure: executing complex aggregations on terabytes of historical data using only a local machine constrained to 8GB RAM, without compromising on reliability or data freshness (T+1 SLA requirement).

The Engineered Solution

I architected an Automated ELT Pipeline that overcomes hardware limitations through a sophisticated Hybrid Architecture combining Python and MySQL.

The system utilizes Python streaming for memory-efficient extraction from SGX data sources and strategically offloads heavy transformations to MySQL Stored Procedures. Raw tick data is structured into a high-performance Star Schema Data Warehouse with yearly partitioning.

Result: A robust analytical engine that successfully powers a visualization dashboard, enabling the detection of institutional "Whale" activity and monitoring intraday price action reliably every morning at 7 AM with full idempotency and data quality checks.

Key Lessons Learned

üí° The Power of Hybrid Architectures

Learned that "one tool doesn't fit all." Combining Python's flexibility for extraction with a Relational Database's power for set-based transformations (SQL) yielded the best performance-to-resource ratio in constrained environments.

üí° Constraints Drive Innovation

The 8GB RAM limit forced a deeper understanding of memory management, streaming algorithms (map-reduce style), and efficient database schema design (Star Schema & Partitioning) that I wouldn't have gained with unlimited cloud resources.

üí° Production vs. "Just Working"

Moving beyond code that just runs to building a system that is robust, monitorable (logs/SLAs), and resilient to failures (idempotency/retries) is the core distinction between engineering and scripting.
